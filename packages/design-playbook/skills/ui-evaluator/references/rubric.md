# evaluator rubric

## 维度如何选

- **落地页**：设计质量 / 原创性 / 工艺 / 功能
- **控制台**：设计质量 / 可用性 / 信息密度 / 工艺 / 一致性

维度从场景读出，引擎不写死；**判据必须能回到声明**。

## 回流示例

| 发现 | 表层说法 | 回指 |
| --- | --- | --- |
| 失败无重试 | 交互不完整 | `spec` 状态流 |
| 高危灰标签 | 颜色不醒目 | `domain` + `components` |
| 按钮硬编码 hex（如 `#4f46e5`） | 代码不规范 | `design` token |
| 列表变卡片墙 | 版式不对 | `template` |

## 禁止

- 「整体还可以优化」无指征
- 只改 CSS 不回到声明
- 用新审美词覆盖未写清的 L5
