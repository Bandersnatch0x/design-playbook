# v0.3.1 - Dedup & single-source

**Tag:** v0.3.1 · **Previous:** v0.3.0 · **Date:** 2026-07-20

## Summary

Patch release: **no behavior change, no surface removed.** Internal de-duplication and single-source cleanup from the wayfinder `dedup-single-source` effort (5 tickets, all resolved via 4-role round-tables for the grilling ones), plus two boundary ADRs.

## Changes

- **Dedup-01 (stdio framing):** extracted shared stdio JSON-RPC framing to `packages/design-playbook/mcp/_transport.py`; preview + evidence servers import `read_message` / `write_message`. Behavior-preserving — the per-server parse-error policy asymmetry (preview crashes, evidence replies -32700 and continues) is documented in `_transport`, not unified. Net -94 lines.
- **Dedup-02 (check overlap):** ruled **not** to extract `scripts/_checks.py` (~18-line overlap doesn't justify a new module). Pinned the validate / release / doctor boundary in `release-checklist.md` "Validation surfaces" + `Mirrors ...` comments on `doctor.py` + docstring cross-refs.
- **Dedup-03 (STAGES dual-source):** two-way pointer between `run_status.py` `STAGES` and SKILL.md Steps (not a lockstep test — ADR-0010 P1 navigation surface is unscheduled, YAGNI). CONTEXT.md Layout splits `.scratch/` into run artifacts (`.scratch/<run>/`) vs decision maps (`.scratch/<effort>/`).
- **Dedup-04 (MCP config):** collapsed the 4 verbatim JSON blocks in the sibling READMEs to pointer paragraphs (plugin `.mcp.json` is the source of truth). Server launch config now lives in exactly 4 places (2 `.mcp.json` + 2 `mcp.example.toml`). `screenshot-showcase.mjs` footer version now reads `plugin.json`. Net -51 lines.
- **Dedup-05 (launcher sunset):** compatibility launchers retained through v0.4.x; evaluated for sunset at v0.5 (ADR-0009 §3, reopen triggers: user dependency signal / CI coverage loss / maintenance burden). No deprecation notice shipped this release — patch discipline, and a notice before a sunset is decided would be an empty commitment.
- **ADR-0010:** input/navigation signals (attention / viewport evidence, structured navigation) may enter only as 2D declaration/evidence extensions; dedicated-hardware input and new runtime gates are non-goals.
- **Wayfinder:** `dedup-single-source` effort registered and closed (`.scratch/dedup-single-source/map.md`, local).
