# v0.4.1 - Control-bar i18n hardening + polish

**Tag:** v0.4.1 · **Previous:** v0.4.0 · **Date:** 2026-07-21

## Summary

Patch: control-bar i18n JS injection hardened (design B - JSON script, removes the `.format()` x JS-string bug where translations containing `"`/`{` broke JS or raised KeyError) + LOW polish, all reviewed and landed. Implemented by a grok worker, reviewed by code-reviewer (APPROVE, no CRITICAL/HIGH).

## Changes

- **#1 i18n JS injection (design B):** control.py JS strings no longer interpolate `{t_xxx}` via `.format()`. Now JSON-injected: `<script>window.DPB_I18N = {i18n_json};</script>` (`json.dumps`, JS-safe), JS reads `window.DPB_I18N.key`. HTML `{t_xxx}` retained (HTML escape safe) with `html_lib.escape` on attribute contexts.
- **#2 dialog.show() fallback:** `.is-open dialog.dpb-drawer{display:flex}` - drawer reveals even if `show()` fails.
- **#3 non-modal focus trap:** Tab trapped when `pinOff`; `pinOn` lets Tab reach the prototype (pin needs element selection).
- **#4 abort announce:** 4s timeout broadcasts "cancelled" to the sr-only `#dpb-abort-status` (a11y).
- **#5 dead code:** removed the 3rd `.replace` in `_build_control` (dead after the 2nd fully replaced the class string).
- **i18n.py:** +`locate` / `abort_cancelled` keys (zh/en).
- **code-reviewer polish (all fixed in this release):** HTML attribute placeholders escape `quote=True` (aria-label/placeholder); `json.dumps` `</script>` defense; `abort_cancelled` merged into the `JS_KEYS` tuple.
