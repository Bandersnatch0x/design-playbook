# v0.4.1 - Control-bar i18n hardening + polish

**Tag:** v0.4.1 · **Previous:** v0.4.0 · **Date:** 2026-07-21

## Summary

Patch: control-bar i18n JS injection hardened (design B — JSON script, kills the `.format()` × JS-string坑) + LOW polish. Implemented by grok worker, reviewed by code-reviewer (APPROVE, no CRITICAL/HIGH).

## Changes

- **#1 i18n JS injection (design B):** control.py JS strings no longer interpolate `{t_xxx}` via `.format()` (translations with `"`/`{` broke JS or raised KeyError — MEMORY `format-on-html-css-braces`). Now JSON-injected: `<script>window.DPB_I18N = {i18n_json};</script>` (json.dumps, JS-safe), JS reads `window.DPB_I18N.key`. HTML `{t_xxx}` retained (HTML escape safe) + `html_lib.escape` added for attribute contexts.
- **#2 dialog.show() fallback:** `.is-open dialog.dpb-drawer{display:flex}` — drawer reveals even if `show()` fails.
- **#3 non-modal focus trap:** Tab trapped when `pinOff`; `pinOn` lets Tab reach prototype (pin needs element selection).
- **#4 abort announce:** 4s timeout broadcasts "cancelled" to sr-only `#dpb-abort-status` (a11y).
- **#5 dead code:** removed 3rd `.replace` in `_build_control` (after 2nd fully replaced the class string).
- **i18n.py:** +`locate` / `abort_cancelled` keys (zh/en).
