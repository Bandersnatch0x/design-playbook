## Summary

This release focuses on perfecting the v0.3 scope based on Q3 grill decisions and DX feedback triage:

- **Preview control bar UX overhaul** (control.py):
  - Pill primary is now context-sensitive: direct submit (confirm) when readiness is met (substantive feedback or complete anchors), no forced drawer open.
  - Added "保留批注，暂不决定" (keep notes, decide later) draft button in drawer footer.
  - Label auto-switches to confirm text + `.is-direct-confirm` styling when ready; **restores original label on ready->not-ready flip** (I13).
  - Mobile: revise button no longer hidden.
  - A11y: `aria-haspopup` honest (removed when direct-confirm, restored when drawer-trigger); readiness announcement cached to stop screen-reader jitter (I7).
  - Revise remains lenient (bypasses floor).
- **opencode review P1 pass** (4-round code-review, all green):
  - Abort renamed "终止评审"/"End review", moved to footer left (destructive far from primary), danger tint + **two-step confirm** (arm 2s -> second click submits, resets on drawer close) (I4/I5/I18).
  - Pill revise opens drawer pre-focused instead of one-click naked submit (I3).
  - Scrim intercepts clicks (close-on-outside) when not pinning; pass-through in pin mode (I6).
  - `Ctrl/Cmd+Enter` in feedback submits the confirm (I8); `submitPrimary()` shared by pill-direct + Ctrl+Enter (DRY).
  - Shortened not-ready pill text (I9); deduped `.dpb-round` CSS (I10); removed dead `focusableEls()` (I12).
- **ADR-0008 floor enforcement** fully landed across sites + tests.
- **Test coverage**: `test_floor_frontend.py` S1-S15b (direct confirm, draft, label switch + restore, abort two-step + reset-on-close, Ctrl+Enter action-correctness). All pass.
