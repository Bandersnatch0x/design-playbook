# v0.5.0 - G5 preview integrity hardening + reference-intake

**Tag:** v0.5.0 · **Previous:** v0.4.4 · **Date:** 2026-07-22

## Summary

Minor bump for two feature-grade changes plus a round of preview/evidence
fixes. The G5 preview confirmation gate is hardened against two confirmed
bypass classes (renumber replay + hash-strip) and a serve-time TOCTOU
swap. A new `reference-intake` skill lands (ADR-0011). The `secure-ship-0.4.4`
grilling cycle closes fully (tickets 01–09 resolved).

## Changes

### feat(security): G5 preview integrity (ticket 02/03, `da38edd`)

Round-table adjudication (factual/security/engineer/consistency) closed two
Critical G5 bypasses found in the HEAD validator + a High TOCTOU:

- **BYPASS-1 (renumber replay)**: `_prototype_target` no longer trusts the
  record's `prototype_path` to select the hash target. The target is always
  `preview/round-<latest>.html`, derived from the latest numeric round.
  Round cross-check: filename round must equal the JSON `round` field.
- **BYPASS-2 (hash-strip)**: a confirm record missing `prototype_html_hash`
  now FAILs (the post-0.4.4 adapter always writes it; absence = hand-written
  or pre-0.4.4 record).
- **TOCTOU swap**: `browser.py` reads prototype bytes once at serve time,
  hashes, and decodes for display; the digest flows to `_write_confirm`
  (which no longer re-reads disk). Served bytes == hashed bytes.
- **fail-closed + single source**: `is_confirmed_valid` now rejects
  `aborted:true`; `latest_numeric_round` unions `log.md` round headings
  (deletion-rollback protection).

### feat(reference): reference-intake skill (ADR-0011, `5767346`)

New skill for intake of design references into the SSOT references tree.

### fix(preview): pin-to-annotate bridge restored (`7a02dc3`)

Restored the postMessage bridge for pin-to-annotate across the sandboxed
iframe (G5 trust-boundary regression).

### fix(g6): casefold evidence/ prefix (`4d92766`)

Canonicalized the lowercase `evidence/` prefix on artifact path resolve.

### fix(preview): hashlib regression (`c107039`)

Hotfix for a regression introduced by the 02/03 TOCTOU change: `browser.py`
used `hashlib` without importing it, and `_write_confirm` callers/tests still
used the old `prototype=` signature. Added the import, a serve-fallback in
`server.py`, and updated the test signature.

## Known limitations

- **Codex RUN_ROOT is opt-in**: the host must set `DESIGN_PLAYBOOK_RUN_ROOT`
  before launching codex, else evidence artifacts fall back to the plugin
  cache root. Codex has no host-workspace variable (verified in source,
  rust-v0.142.5). `written_path` is returned to the orchestrator so
  misconfiguration is visible. Not a security hole — G6 write boundary holds.
- **Frontend floor**: `test_floor_frontend.py` (ADR-0008 JS verification) is
  a documented skip-with-reason in CI, not yet a required gate.
