# v0.4.4 - Codex marketplace + re-review security hardening

**Tag:** v0.4.4 · **Previous:** v0.4.3 · **Date:** 2026-07-21

## Summary

Security hardening release closing the v0.4.3 re-review Critical/High
findings (verdict Recirculate -> repaired) plus the Codex-native marketplace
install path shipped on main after v0.4.3. The G5 preview trust boundary is
now physically isolated (sandboxed iframe + one-time server token), G6
evidence writes are confined to the `evidence/` subtree, the state machine
reuses the validator and fails closed, and the Codex/Claude dual-publish
surfaces are statically gated.

## Changes

- **G5 trust boundary (Critical):** prototype rendered inside
  `<iframe sandbox="allow-scripts" srcdoc="...">` (no `allow-same-origin`) so
  prototype scripts cannot reach the parent DOM; one-time decision token
  (`secrets.token_urlsafe(32)` + `compare_digest`) bound to the round;
  first-decision-wins session lock (TOCTOU-safe via `threading.Lock`);
  `do_POST` fails closed on missing/mismatched/reused token; anti-DoS - a
  forged tokenless or arbitrary-token POST no longer aborts the session
  (`done.set()` gated on a genuine token match).
- **G6 evidence containment:** `_resolve_artifact_path` forces the
  `<run_root>/evidence/` subtree (rejects `..`, absolute paths, symlink
  escape via `realpath`); `overwrite` is opt-in (default refuses existing
  files); `validate_run.check_evidence` rejects `observed` that escapes
  `evidence/`.
- **validator / state machine:** `check_preview` enforces the latest
  **numeric** round (kills the lexicographic `round-10 < round-2` bug) and
  verifies `prototype_html_hash` (recomputed from disk bytes, not
  self-reported); `run_status` reuses the validator's
  `latest_numeric_round` / `is_confirmed_valid` helpers and fails closed on
  aborted / floor-failed / stale-confirm records (never "resume at fill").
- **dual-publish gates:** `validate.py` / `doctor.py` read the Codex
  manifests (`.codex-plugin/plugin.json` version match, MCP target
  existence, `.agents` marketplace source path); `release.py check_version`
  spans 6 version sites (Claude plugin.json + marketplace meta + marketplace
  plugin + Codex plugin.json + README x2).
- **Codex RUN_ROOT:** dropped the static `DESIGN_PLAYBOOK_RUN_ROOT: "."`
  from both Codex and Claude manifests (ADR-0009 symmetry) so Evidence no
  longer anchors to the plugin install root; `written_path` still reports
  the resolved absolute path for smoke assertion.
- **CI:** Preview / Evidence MCP stdio tests are required gates; Playwright
  install + skip-with-reason (frontend floor deferred with a TODO).
