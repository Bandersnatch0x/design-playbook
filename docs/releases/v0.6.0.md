# v0.6.0 - design-baseline gate + G5 LF-normalized hash

**Tag:** v0.6.0 · **Previous:** v0.5.0 · **Date:** 2026-07-24

## Summary

Minor bump for existing-product visual baseline initialization (ADR-0012) plus a
cross-platform G5 integrity fix. New conditional skill `design-baseline?` binds
or drafts a project-owned `DESIGN.md` before Fill. Prototype HTML hashes now
normalize line endings so Windows `core.autocrlf` checkouts match Linux CI.

## Changes

### feat(design-baseline): existing-product DESIGN.md gate (ADR-0012)

- New skill `packages/design-playbook/skills/design-baseline/` with deep module
  `prepare` / `confirm` / `verify`.
- Sole gate artifact: `.scratch/<run>/design-baseline/state.json`
  (`schema: design-baseline/v1`).
- Statuses: `ready` | `needs_confirmation` | `waived` | `ambiguous`.
- Existing-product Fill requires verified `ready` (path + sha256) or explicit
  `waived` (non-empty reason). Draft alone is not authority.
- Path safety: project-root containment, symlink rejection, control-char
  rejection, bounded state/DESIGN reads (4 MiB), post-write TOCTOU re-check on
  canonical accept.
- Provenance-minimal existing baselines: only frontmatter +
  `Source Evidence & Confidence` required to bind; other sections remain draft
  template guidance.
- Orchestrator / ui-picker / craft-guard / ui-evaluator / run_status / doctor
  skill count (8) wired. Dead `inspect_design_baseline` helper removed.

### fix(preview): LF-normalize prototype_html_hash for G5 integrity

- Shared `prototype_html_digest` (writer + validator) hashes LF-normalized
  prototype bytes so Windows `core.autocrlf` working trees do not disagree with
  Linux CI on the same git blob.
- Fixtures updated; seam test covers CRLF working-tree vs LF-normalized hash.
