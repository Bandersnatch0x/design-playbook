# v0.4.2 - Pill direct-confirm two-step arm

**Tag:** v0.4.2 · **Previous:** v0.4.1 · **Date:** 2026-07-21

## Summary

Patch: the ready-state pill primary ("Confirm") no longer submits on a single
click. An accidental click on a resting confirm button could end the review
session with a false pass; the pill now uses the same two-step arm pattern as
the abort button (I18).

## Changes

- **control.py:** first click arms (pulse ring + "click again to confirm"
  label + sr-only `role=alert` broadcast), second click submits. Arm is undone
  by a 4s timeout, annotate/revise click, drawer open/close, or a readiness
  flip - never submitting as a side effect. `prefers-reduced-motion` disables
  the pulse animation.
- **i18n.py:** +`confirm_confirm` / `confirm_cancelled` keys (zh/en), wired
  into the `JS_KEYS` tuple for the JSON i18n injection (v0.4.1 design B).
- **tests:** S10 reworked to assert arm-then-submit (no submit on first
  click, drawer stays closed); S17 added (arm undo via annotate click and via
  4s timeout, with no submit either way).

## Validation

FRONTEND FLOOR TEST (S1-S17), SEAM TEST, `release.py` dry-run - all PASSED.
